Ils sont mis à disposition par l’université Irvine, il s’agit de datasets réels, vous trouverez généralement sur chaque page l’article que ces données ont permis de publier, ce qui peut vous aider à comprendre le contexte, la manière dont les données sont collectées, etc.

Vous pouvez bien sûr chercher d’autres jeux de données.

Pour le projet vous conduirez une EDA sur un jeu de données que vous sélectionnerez.

Je vous recommande de choisir le jeu de donnée assez tôt (vous pouvez essayer plusieurs jeux de donnnées). Après avoir lu le cours et traité les exemples données, vous devez reproduire les traitements montrés sur ces données, ce qui vous permettra de prendre de l’avance sur votre projet et de tester vos compétences au fur et à mesure que vous avancez dans le cours.

Il est attendu que vous démontriez dans votre exposé/notebook les compétences que vous aurez acquises :

- compréhension des données (exposer le contexte - connaissance métier, problématique… –, quelles sont les données, leur type, leur mode de collecte, etc.)
- préparer les données pour les analyses ultérieures
- vérifier la qualité de la donnée ( valeurs manquantes ? outliers? si on en trouve comment les gère-t-on ?)
- créer des visualisations pertinentes qui donne une idée de votre compréhension des données
- identifier les tendances, formuler des hypothèses, justifier vos choix (pourquoi tel modèle ? pourquoi telle hypothèse ?)
- conduire des analyses mettant à jour des relations entre les variables
- interpréter les résultats des analyses (les inférences statistiques, la significativité, etc.)
- conclure en soulignant les limitations d’une part, les perspectives d’autre part
- on tiendra compte de la qualité générale du notebook : clarté, rigueur dans l’écriture du code et les analyses/conclusions, structuration de l’analyse et du plan, facilité de lecture, etc.
